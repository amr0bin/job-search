# Cover letter library

## How this works

This library contains modular cover letter versions by sector. Each file is a complete, tailored letter.
When applying, open the closest sector version, duplicate it to `applications/sent/`, and customize.

**Git handles versioning** — you never need to name files `v2-final-revised.docx`.
To see the history of any file: `git log --follow -p library/cover-letters/higher-ed.md`

## Customization checklist (for each application)

Before sending any letter, confirm:

- [ ] Org name spelled correctly throughout
- [ ] Hiring manager name used if known (check LinkedIn)
- [ ] Correct sector vocabulary used (see `strategy/sector-vocabulary.md`)
- [ ] Most relevant case study mentioned by theme (not by name — they're anonymized)
- [ ] Salary/remote preference NOT mentioned (save for interview unless posting asks)
- [ ] Closing paragraph references something specific about this org
- [ ] Saved to `applications/sent/YYYY-MM-orgname-roletitle.md`
- [ ] Airtable row updated

---

## Length and format guidance

- **Length:** 3–4 paragraphs. Never more than one page.
- **Opening:** Don't start with "I am writing to apply." Start with a statement about the org or the problem they're trying to solve.
- **Paragraph 2:** What you bring — specific, with reference to a relevant project.
- **Paragraph 3:** Why this org specifically — something you know about them that most applicants won't.
- **Closing:** Clear, confident, not sycophantic.
