Each file is a complete resume version tailored to a sector.
Store as markdown — convert to PDF for sending using your preferred tool (Docs, Word, etc.).
Git tracks every change, so you always have the history.

## The master-content principle

`master-content.md` is your single source of truth — every role, every achievement, every skill.
Sector versions are edited-down subsets: same content, different emphasis and vocabulary.

**Never edit sector versions directly without updating master-content first.**

## ATS and format notes

- BC public sector uses an online application system — paste resume content into fields, don't rely on PDF formatting
- Universities often use Workday or PeopleSoft — same issue
- Research orgs and cultural orgs usually accept PDF — use your formatted version
- Keep formatting clean: no tables, no text boxes, no headers/footers with key info
- Every version should be scannable in 30 seconds: clear section headers, bullet points for roles, quantified outcomes where possible
